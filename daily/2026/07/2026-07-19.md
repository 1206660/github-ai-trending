### 1. tirth8205/code-review-graph 📈 +355 stars (总计 20,173)

- **简介**：本地优先的代码智能图谱（Code Intelligence Graph）工具，通过 Tree-sitter 构建代码的结构化依赖关系图，让 AI 编程助手（Claude Code、Cursor、Codex 等）在代码审查时只读取最小必要上下文，减少 Token 浪费。支持 MCP 协议。
- **增长原因**：昨日（2026-07-18）发布 v2.3.7，这是一次维护者整合（maintainer-reconciliation）版本。项目非常活跃（每天有提交），安全修复和社区贡献持续合并。其定位精准解决 AI 编码工具 Token 浪费的痛点，在 Agent/MCP 生态日益兴起的背景下需求量大。
- **核心特点**：
  - 基于 Tree-sitter 语法解析，支持 30+ 编程语言（包括 Jupyter Notebook）
  - 增量更新 <2 秒，可配置 Git hooks 或 watch 模式自动更新
  - 爆炸半径（Blast-radius）分析：文件变更时追踪所有调用链、依赖关系和测试覆盖
  - 测评显示每问题 Token 减少中位数 ~82 倍（最高 528 倍）
  - 支持语义搜索（向量嵌入）、社区检测（Leiden 算法）、图可视化（D3.js）
  - 提供 GitHub Action，可在 PR 中自动生成风险评分的代码审查评论
  - 支持自定义语言：通过 `.code-review-graph/languages.toml` 添加新语言（无需 fork）
- **应用场景**：使用 AI 编程助手进行日常开发的团队和个人，特别是大型 Monorepo 项目——可将 27,700+ 文件的审查上下文压缩到 ~15 个关键文件。
- **风险/局限**：
  - 小变更场景下图上下文可能超过直接文件读取（overhead）
  - 召回率 1.0 来自图推导的上界，实际准确率约 0.71 F1
  - 流检测（Flow detection）仅 33% 召回率（JavaScript/Go 较弱）
  - 构建首图需要一次全量解析（500 文件约 10 秒）

### 2. KnockOutEZ/wigolo 📈 +203 stars (总计 1,243)

- **简介**：为 AI 代理打造的本地优先 Web 智能层——无需 API Key、无需云服务、零成本。提供搜索、抓取、爬取、结构化提取、缓存和研究等功能，通过 MCP 协议集成到 Claude Code、Cursor、Codex、Gemini CLI 等 AI 编码工具中。
- **增长原因**：2026-07-17 发布 v0.2.0 版本，近期非常活跃（每天 5-10 次提交）。其 "零成本、零 API Key" 的定位在 AI Agent 生态中极具吸引力，与 Firecrawl、Tavily、Exa 等付费服务直接竞争但完全免费。社交传播效应显著。
- **核心特点**：
  - 18 个搜索引擎适配器并行搜索，结果融合排序（rank fusion + ML reranking）
  - 自动降级加载：HTTP → headless browser（遇到反爬虫时自动切换）
  - 多页爬取支持 BFS/DFS/Sitemap 模式，有域名限速和 robots.txt 遵守
  - 结构化提取：表格、JSON-LD、元数据、自定义 JSON Schema
  - 本地缓存：所有抓取数据存储在本地，可重新查询（keyword + hybrid semantic）
  - 自主学习/自治代理模式：分解问题 → 分发子查询 → 合成有引用的报告
  - 支持 MCP Server + REST API + SDK（TypeScript/Python）+ CLI 四种接入方式
- **应用场景**：AI 编程助手需要实时网络数据时（查文档、找解决方案、调研竞品），且不希望产生 API 调用费用的开发者和团队。特别适合高频调用场景。
- **风险/局限**：
  - 项目处于 Public Beta 阶段（v0.2.0），稳定性有待验证
  - 需要 Node.js ≥20 和 ~1.5GB 磁盘空间存放浏览器引擎和本地模型
  - 自主研究模式（research/agent）需要 LLM API Key（建议免费 Gemini Key）才能获得最佳体验
  - 许可证为 AGPL-3.0，对商业嵌入有约束

### 3. lyogavin/airllm 📈 +161 stars (总计 23,337)

- **简介**：AirLLM 通过层分解（Layer-wise decomposition）技术，大幅降低大模型推理内存需求——70B 模型只需单张 4GB GPU，405B Llama 3.1 可在 8GB 上运行，DeepSeek-V3（671B）只需 ~12GB。无需量化、蒸馏或剪枝。
- **增长原因**：2026-06-30 发布 v3.0.0 大版本更新，增加了 FP8 模型支持和最新模型兼容（DeepSeek-V3、Qwen3-235B 等），Star 持续稳定增长。每日 Star 历史图显示匀速上涨趋势，是 GitHub Trending 的常客。
- **核心特点**：
  - 单卡运行超大规模模型：70B/4GB, 405B/8GB, 671B/12GB
  - Block-wise 量化压缩：推理速度提升 3 倍，精度损失可忽略
  - AutoModel 自动检测模型类型，无需手动指定
  - 支持 CPU 推理（非 GPU 用户也可使用）
  - 支持 macOS（MacBook 上运行 70B 模型）
  - 兼容主流模型：Llama 3/4, Qwen3, DeepSeek V2/V3, Phi-4, Gemma, ChatGLM, Mistral 等
- **应用场景**：GPU 预算有限的 AI 研究者、学生和爱好者，希望在自己的硬件上运行大模型进行实验和推理的个人开发者。
- **风险/局限**：
  - 推理速度比原生加载慢（层分解需要多次 I/O 操作），有压缩约 3x 加速
  - 需要足够的磁盘空间存储分层模型（原始模型 → 分解后的层保存）
  - 项目主要维护者为个人开发者（lyogavin），长期维护持续性存疑
  - 最新大版本 v3.0 发布已近三周，近期提交仅为文档更新，无实质性代码变更

### 4. elder-plinius/G0DM0D3 📈 +69 stars (总计 9,511)

- **简介**：开源的多模型 AI 聊天接口，专注于 Red Teaming（红队测试）和认知研究。支持 60+ OpenRouter 模型和本地模型，提供 GODMODE CLASSIC（5 种提示+模型并行竞赛）和 ULTRAPLINIAN（多模型评估引擎）等特色功能，以及 33 种输入扰动技术（Parseltongue）用于测试模型鲁棒性。
- **增长原因**：项目持续活跃开发（7 月 14-15 日有功能提交）且在 AI 安全和红队社区中口碑良好。新加入的本地模型支持（Ollama/LM Studio/llama.cpp/vLLM）扩大了受众范围。ULTRAPLINIAN 多模型评估引擎是独特卖点。
- **核心特点**：
  - GODMODE CLASSIC：5 个模型+提示组合并行对比，选出最佳回复
  - ULTRAPLINIAN：多模型比较评估引擎，5 个难度等级（12-60 个模型同时竞赛），100 分复合评分
  - Parseltongue：33 种输入扰动技术（Leetspeak、盲文、摩斯码、Unicode 替换等），3 个强度等级
  - AutoTune：根据 20 种查询上下文自动选择采样参数（temperature/top_p/top_k）
  - 支持 OpenRouter、Venice 和本地模型（Ollama/LM Studio/vLLM）
  - 隐私透明：元数据遥测可选，本地模式无日志
  - 单文件 HTML 部署（index.html），可通过 GitHub Pages/Cloudflare Pages 零成本托管
- **应用场景**：
  - AI 安全研究者：进行模型红队测试和鲁棒性评估
  - 认知科学研究：研究模型行为边界和偏好
  - 黑客/技术爱好者：探索 AI 模型的"解放"交互
- **风险/局限**：
  - 项目核心功能（GODMODE）本质上是越狱/对抗性技术，可能存在伦理和使用合规争议
  - 依赖第三方 API（OpenRouter/Venice），无法保证模型始终可用
  - 单文件 HTML 架构功能有限，高级功能（如贡献数据集）在独立页面不可用
  - 遥测系统虽然可选且透明，但默认开启（元数据收集）

### 5. MoonshotAI/kimi-cli 📈 +65 stars (总计 9,493)

- **简介**：月之暗面（Moonshot AI）推出的终端 AI 编程代理，可直接在终端中完成软件开发和 Shell 操作任务。支持代码读写、Shell 命令执行、网络搜索，并能自主规划执行步骤。
- **增长原因**：2026-07-16 发布 v1.49.0 版本，项目持续迭代。虽已宣布将演进为 Kimi Code CLI（下一代终端 AI 代理），但现有用户群体依然稳定。月之暗面（Kimi）品牌在国内开发者中有较强号召力。
- **核心特点**：
  - Shell 命令模式：Ctrl-X 切换，不离开 CLI 即可执行 Shell 命令
  - VS Code 扩展集成：与 Visual Studio Code 深度整合
  - ACP（Agent Client Protocol）支持：可与 Zed、JetBrains 等 IDE 配合使用
  - MCP 支持：支持 Streamable HTTP 和 stdio 两种 MCP 传输方式
  - Zsh 插件集成：增强终端 AI 体验
  - 中文文档完善，开发者体验优秀
- **应用场景**：需要在终端中直接使用 AI 辅助编程的开发者，特别是 Kimi 生态用户和中文开发者。也适合需要 IDE 无关的 AI 编码助手的场景。
- **风险/局限**：
  - 项目正逐步迁移到 Kimi Code CLI，长期维护方向不确定
  - 需要月之暗面的 API Key（国内服务，海外用户有网络延迟限制）
  - 内置 Shell 命令有限（不支持 cd 等基本命令）
  - 依赖 Kimi 后端 API，离线场景无法使用
