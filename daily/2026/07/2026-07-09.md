## 今日 Top Stars 增长榜

### 1. Diolinux/PhotoGIMP 📈 +1125 stars (总计 15,037)
- **简介**：让 GIMP 3+ 在界面布局、快捷键和工作流程上模仿 Adobe Photoshop 的免费社区补丁。Photoshop 用户迁移到 GIMP 的"桥梁"工具。
- **增长原因**：虽然最近一次大版本发布 (v3.0) 是 2025 年 3 月，但 PhotoGIMP 作为 Photoshop 替代方案的认知度持续积累。可能是被某科技媒体/YouTube 频道再次推荐，叠加 GIMP 3 的持续用户增长效应。今日骤增 1125 星表明有大规模曝光事件。
- **核心特点**：Photoshop 风格的工具布局、自定义启动画面、最大化画布空间、Photoshop 键盘快捷键映射、GPL-3.0 开源。
- **应用场景**：从 Adobe Photoshop 迁移到 GIMP 的设计师/摄影师；不想支付 Photoshop 订阅费用的用户；Linux 用户寻找原生图像编辑方案。
- **风险/局限**：依赖 GIMP 版本更新，GIMP 3.x 的小版本更新可能破坏兼容性；仅更改 UI/UX，不扩展 GIMP 本身的功能；项目最近一次更新是 1 周前，维护频率不高。

### 2. TencentCloud/TencentDB-Agent-Memory 📈 +318 stars (总计 7,650)
- **简介**：腾讯云开源的 AI Agent 长期记忆系统，通过 4 层渐进式流水线实现完全本地化、零外部 API 依赖的长期记忆管理。
- **增长原因**：AI Agent 记忆是一个热门赛道 (mem0, mempalace 等竞品)。项目本身是 Hermes Agent 和 OpenClaw 的官方记忆插件，且持续更新（最新 v0.3.6 于 2 个月前发布，最近 2 周仍有代码提交）。开源社区的关注度+腾讯的品牌效应驱动稳定增长。
- **核心特点**：符号化短期记忆（将工具日志压缩为 Mermaid 符号，减少 Token 消耗）+ 分层长期记忆（将零散对话提炼为结构化人物和场景）；零外部 API 依赖；支持 OpenClaw 和 Hermes Agent 集成；全 TypeScript 实现。
- **应用场景**：需要持久化记忆中上下文的 AI Agent 开发者；构建长期对话助手或个性化 AI 应用；团队需要在本地部署记忆系统保护数据隐私。
- **风险/局限**：与腾讯云生态绑定较深；264 个未解决的 Issue 显示项目仍处于快速迭代期，稳定性有待验证；依赖 Node.js >= 22.16，版本要求较高。

### 3. huxingyi/autoremesher 📈 +296 stars (总计 2,016)
- **简介**：跨平台的自动四边形网格重拓扑工具，将高多边形 3D 模型转换为干净的四边形拓扑结构，适用于 3D 建模和游戏开发。
- **增长原因**：**3 天前刚刚发布了 v1.0.0 正式版！** 项目自 2020 年起历经 6 年、8 个预发布版本 (alpha/beta)，终于达到了 1.0 里程碑。大量等待稳定版的 3D 艺术家和开发者开始在今天关注和星标该项目。
- **核心特点**：基于 Geogram、libigl 等成熟库构建；支持 C++14 + Qt 5 跨平台编译；自动生成高质量的四边形网格；提供命令行和图形界面两种使用方式。
- **应用场景**：3D 建模师 (Blender/Maya/3ds Max 用户) 的网格重拓扑；游戏开发中的模型优化；计算机图形学研究；逆向工程中的网格清理。
- **风险/局限**：仅 2,016 星，属于小众专业工具；依赖 Qt 5.15.2 和特定构建环境，不提供预编译二进制包；原始 2020 年 beta 之后社区贡献较少，主要靠作者单人维护。

### 4. prisma/prisma 📈 +46 stars (总计 46,557)
- **简介**：Node.js & TypeScript 的下一代 ORM，支持 PostgreSQL、MySQL、MariaDB、SQL Server、SQLite、MongoDB 和 CockroachDB 等多种数据库。
- **增长原因**：持续维护的成熟项目。最新 v7.8.0 发布于 4 月，近期 Pulse 显示有活跃的 PR 活动（包括 "agent-native initiative"、Prisma Agent Skills 等新方向）。今日 +46 星是稳定大项目的日常有机增长。
- **核心特点**：声明式数据模型（Prisma Schema）；自动生成类型安全的客户端；强大的迁移工具；支持关系数据库和文档数据库；活跃的社区和生态。
- **应用场景**：TypeScript 全栈开发；Node.js API 后端；微服务架构中的数据访问层。
- **风险/局限**：2,651 个未解决的 Issue 数量庞大；MongoDB 支持不如关系型数据库成熟；ORM 抽象层存在 N+1 查询等经典问题。

### 5. argoproj/argo-cd 📈 +29 stars (总计 23,434)
- **简介**：Kubernetes 的声明式持续交付工具，遵循 GitOps 理念，通过 Git 仓库管理应用部署和生命周期。
- **增长原因**：上周发布了 v3.5.0-rc2 候选版。Kubernetes 社区对 GitOps 工具的需求持续增长，新的候选版本吸引了存量用户回归关注。
- **核心特点**：基于 Git 仓库的声明式部署；自动同步和应用状态修正；支持多集群管理；Web UI 和 CLI 双界面；OIDC/OAuth2 集成；丰富的 RBAC 权限控制。
- **应用场景**：Kubernetes 运维团队；DevOps 工程师实施 GitOps 工作流；多云/多集群部署管理。
- **风险/局限**：7,425 个 Fork 和 4,235 个 Issue 说明社区规模庞大但也有大量问题；RC 版本尚未正式发布；配置复杂度较高。

- **简介**：Claude 的 MCP 服务器，通过 Model Context Protocol 为 Claude 提供终端控制、文件系统搜索和差异化文件编辑能力。
- **增长原因**：MCP (Model Context Protocol) 生态持续火热。该项目版本迭代非常活跃（v0.2.43 于 2 周前发布），最新提交仅 17 小时前。作为 Claude 桌面扩展的重要工具，受益于整个 MCP/AI Agent 工具生态的增长。

- **应用场景**：希望通过 Claude 直接操作文件系统和终端的开发者；Claude Code/Claude Desktop 的重度用户；需要 AI 辅助进行代码编辑和项目管理的场景。
- **风险/局限**：给予 AI 终端权限存在安全隐患（项目通过用户交互确认来缓解）；依赖 Claude Desktop 和 MCP 协议；120 个未解决 Issue；项目仍处于快速迭代期 (v0.x)，API 可能变化。
