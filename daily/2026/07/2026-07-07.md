### 1. firecrawl/firecrawl 📈 +867 stars（总计 146,267）

- **简介**：开源的 API 服务，专为 AI/LLM 工作负载设计的 Web 搜索、抓取和数据交互引擎。可将任意 URL 转换为干净的 Markdown 或结构化 JSON，自动处理 JS 渲染、反爬、限流等复杂问题。
- **增长原因**：v2.11.0 两周前发布，带来多项重磅功能：Firecrawl Research Index（搜索 300 万+ arXiv 论文）、Keyless 访问（无需 API Key）、自动 PII 脱敏、确定性 JSON 格式。过去一周 21 次 commit、38 个合并 PR，开发极其活跃。
- **核心特点**：覆盖 96% 网页（包括 JS 重型页面）；LLM-ready 输出（Markdown/JSON）；内置旋转代理和反爬处理；MCP 协议支持；Python/Node/Java/Rust/Go SDK 全覆盖；Agent 端点可自然语言驱动数据采集。
- **应用场景**：AI Agent 的 Web 访问层、RAG 内容管道、训练数据采集、竞争情报监控、电商数据提取、企业合规数据采集。
- **风险/局限**：AGPL-3.0 强版权协议；自托管部署复杂（需 PostgreSQL、Redis、浏览器引擎）；大规模使用云服务成本较高；LLM 驱动的功能依赖外部 API。

### 2. bradautomates/claude-video 📈 +427 stars（总计 4,242）

- **简介**：为 Claude 和 Codex、Cursor、Copilot 等 AI 编程 Agent 添加「看视频」能力的技能包。用户粘贴视频 URL 并提问，工具自动下载、提取帧、转录字幕，将完整多模态上下文交给 AI Agent。
- **增长原因**：v0.2.0 上周发布，重构为自包含 Agent Skills 包，修复 Codex 兼容性，支持 50+ AI 编程 Agent 宿主。项目处在大爆发早期阶段，YouTube/自媒体传播推高热度。
- **核心特点**：支持 YouTube、Loom、TikTok、Instagram、本地文件等数百种视频源；4 种详细模式（transcript/efficient/balanced/token-burner）；帧去重算法节省 Token；智能帧预算自动根据视频时长调整；双转录通道（字幕优先 + Whisper 兜底）。
- **应用场景**：Bug 复现视频分析、长视频摘要、课程/演讲笔记提取、广告竞品分析、播客内容结构化。
- **风险/局限**：单维护者项目（bus factor 低）；长视频（>10分钟）精度下降；帧分析消耗大量 Token；需要 yt-dlp/ffmpeg 依赖。

- **应用场景**：开发者发现新工具/库的入口、技术栈调研起点、学习路线图参考。

### 4. karakeep-app/karakeep 📈 +199 stars（总计 26,902）

- **简介**：自托管的「一切皆可收藏」应用（原名 Hoarder）—— 书签、笔记、图片、PDF 的全能管理工具，支持 AI 自动标签、全文搜索、内容归档和协作。
- **增长原因**：Pocket 关停驱动用户迁移 + v0.32.0 大版本（AI Agent 集成、Safari 扩展、移动端重构）+ AI Agent 生态爆发对记忆/检索工具的需求。
- **核心特点**：AI 自动标签与摘要（OpenAI + Ollama）；Meilisearch 全文搜索；Monolith 全页面归档防链接失效；yt-dlp 视频归档；Chrome/Firefox/Safari 扩展 + iOS/Android 原生应用；MCP 服务器和 CLI 支持 AI Agent 集成。
- **应用场景**：替代 Pocket/Instapaper 的 Read-it-later 工具、个人知识库、研究资料管理、团队协作收藏、AI Agent 的持久化记忆存储。
- **风险/局限**：自托管复杂度较高（需要 Docker、PostgreSQL、Meilisearch、Puppeteer）；资源占用大；AI 功能依赖 OpenAI API 或本地 Ollama；仍处快速开发期可能有 breaking changes。
