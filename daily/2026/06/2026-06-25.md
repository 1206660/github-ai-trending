### 1. google-labs-code/design.md 📈 +619 stars (总计 ★17,345)

- **简介**：Google Labs 推出的 `DESIGN.md` 格式规范，用于向编程 Agent 描述品牌视觉形象。结合 YAML 前端设计令牌（颜色、字体、间距等）和 Markdown 设计理念说明，让 AI 编码 Agent 能持久、结构化地理解设计系统。
- **增长原因**：Agent 编程生态爆发（Cursor、Claude Code、Codex 等），开发者急需一套标准化的方式向 Agent 传达设计意图。Google 官方出品 + `npx @google/design.md lint` / `diff` 实用工具链加持，精准命中刚需。
- **核心特点**：
  - **双层级结构**：YAML front matter 提供机器可读的设计令牌，Markdown 正文提供人类可读的设计 rationale
  - **内置验证工具**：`lint` 命令检查令牌引用断裂、WCAG 对比度；`diff` 命令检测版本间设计回归
  - **Agent 原生**：Agent 读取后即可生成符合设计系统的 UI，减少人工 prompt 调整
- **应用场景**：前端团队维护设计系统、Agent 驱动的 UI 生成工作流、跨团队设计一致性保障
- **风险/局限**：仍为早期规范，生态工具链尚在发展中；需要团队维护 `DESIGN.md` 文件本身

### 2. stablyai/orca 📈 +331 stars (总计 ★6,801)

- **简介**：一款桌面 + 移动端的 AI 编排器（ADE - Agent Development Environment），让开发者同时运行 Codex、Claude Code、OpenCode、Pi 等多个 Agent，每个 Agent 在独立的 git worktree 中工作，统一管理。
- **增长原因**：多 Agent 并行工作流是当前热点，Orca 的 "Parallel Worktrees" 功能——一个 prompt 并行发给 5 个 Agent 对比结果——极具吸引力。YC 背书、iOS 移动端监控、Ghostty 级终端渲染等亮点密集。
- **核心特点**：
  - **并行 Worktrees**：一次 prompt 分发给多个 Agent，在隔离 worktree 中独立运行，对比择优
  - **移动端伴侣**：iOS/Android 应用，可在手机上监控 Agent 执行进度并发送后续指令
  - **Design Mode**：点击真实 Chromium 窗口中的 UI 元素，将 HTML/CSS/截图直接送入 Agent prompt
  - **GitHub & Linear 原生集成**：在应用内浏览 PR、Issue、看板
- **应用场景**：重度 AI 辅助开发者、多 Agent 工作流探索者、需要在移动端遥控 Agent 的开发者
- **风险/局限**：桌面端应用生态竞争激烈（Cursor、Windsurf 等），需持续差异化；多 Agent 并行对硬件资源要求较高

### 3. interviewstreet/hiring-agent 📈 +203 stars (总计 ★2,183)

- **简介**：InterviewStreet（HackerRank 母公司）开源的 AI 简历评估 Agent，将 PDF 简历解析为结构化数据，结合 GitHub 信号，输出公平、可解释的评分报告。
- **增长原因**：AI 辅助招聘是 HR Tech 热点。InterviewStreet/HackerRank 在技术招聘领域有品牌信任度。支持完全本地运行（Ollama），对隐私敏感的企业用户吸引力大。
- **核心特点**：
  - **端到端 Pipeline**：PDF 解析 → LLM 结构化提取 → GitHub 信号增强 → 严格评分（含加分/扣分）
  - **本地优先**：支持 Ollama 本地模型，无需将简历数据发送到外部 API
  - **可解释评分**：每项评分有证据支撑，附带 fairness 约束
  - **Pydantic 模式校验**：从 LLM 自由文本中提取结构化数据并规范化
- **应用场景**：HR/招聘团队自动化简历初筛、技术面试前候选人评估、企业招聘数据隐私保护
- **风险/局限**：LLM 提取的准确性依赖模型质量；GitHub 信号可能有偏差（部分优秀开发者没有活跃 GitHub 活动）

### 4. kunchenguid/no-mistakes 📈 +110 stars (总计 ★2,133)

- **简介**：一款 Git 代理工具，在本地 git remote 前加一层质量门禁。`git push no-mistakes` 后会创建一个隔离 worktree，运行 AI 驱动的验证流水线（Code Review → 测试 → 文档 → Lint → Push → PR → CI），全部通过后才提交到真正的远程仓库并自动开 PR。
- **增长原因**：Agent 生成的代码量暴增，开发者需要自动化质量把关。`"Kill all the slop. Raise clean PR."` 的定位精准，Demo GIF 视觉冲击力强。支持所有主流编码 Agent（claude, codex, opencode, pi 等）。
- **核心特点**：
  - **非阻塞**：流水线在隔离 worktree 中运行，不影响当前工作
  - **Agent 无关**：支持多种编码 Agent，通过 `/no-mistakes` 指令让 Agent 自动触发热门
  - **智能修复**：安全的机械修复自动应用，涉及意图的修改升级给开发者审批
  - **一键 PR**：所有检查通过后自动创建干净的 PR
- **应用场景**：使用 AI 编码 Agent 的开发者/团队、需要代码质量自动化门禁的项目、CI/CD 前预防性检查
- **风险/局限**：对复杂项目的完整流水线可能较慢；AI 驱动的 code review 质量依赖底层模型；需要信任 Git 代理层
