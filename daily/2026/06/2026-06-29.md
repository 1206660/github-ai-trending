### 1. Robbyant/lingbot-map 📈 +372 stars（总计 8,229）

- **简介**：LingBot-Map 是一个基于 Transformer 的 feed-forward 3D 基础模型，能从流式数据（如视频）实时重建 3D 场景。本质是 3D 版"视频理解"——输入视频流，输出稠密 3D 几何。
- **增长原因**：论文《Geometric Context Transformer for Streaming 3D Reconstruction》于 4 月登 arXiv，今天在社交网络（X/Twitter、Reddit）上被广泛传播。同时项目提供了 HuggingFace 模型权重和完整演示，降低了复现门槛，吸引大量 CV 研究者试用。
- **核心特点**：**Feed-forward 架构**，无需 test-time optimization（不同于 NeRF/3DGS 需要逐场景训练）；支持 streaming 数据流式重建；单一静态模型覆盖多样化场景；Apache-2.0 开源。
- **应用场景**：机器人实时环境感知、自动驾驶场景重建、AR/VR 空间理解、无人机航拍 3D 建模。
- **风险/局限**：feed-forward 模型泛化能力受训练数据分布限制，极端场景（低光照、极端视角）可能退化；基于 arxiv 论文，生产环境成熟度还需验证。

### 5. cupy/cupy 📈 +174 stars（总计 11,521）

- **简介**：CuPy 是 NumPy/SciPy 的 GPU 加速替代品，提供与 NumPy 几乎一致的 API 接口，但底层运行在 CUDA/ROCm GPU 上，支持 NVIDIA 和 AMD GPU。
- **增长原因**：AI/ML 工作负载持续增长，GPU 加速数值计算需求常态化。CuPy 作为最成熟的 NumPy GPU 版，在新用户中持续获得认可。174 stars 是其稳定的日常增长。
- **核心特点**：**API 兼容 NumPy**——迁移成本极低，`import cupy as cp` 即可；支持 CUDA（NVIDIA）和 ROCm（AMD）双平台；提供 cuBLAS/cuDNN/cuRand 等 CUDA 库的直接接口；支持分布式计算（NCCL 通信）。
- **应用场景**：数据科学家/ML 工程师加速数值计算；深度学习预处理流水线；科学计算（物理学、天文学等）大规模矩阵运算。
- **风险/局限**：需要 NVIDIA/AMD GPU（无 GPU 退路）；部分 SciPy API 尚未覆盖；pip 安装涉及 CUDA 版本匹配，新手容易踩坑（官方推荐 conda-forge）。
